# Report: 타이머기능

## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | 특정 시간대에만 패킷 캡처하려면 사용자가 수동으로 직접 실행해야 했다. |
| **Solution** | 시작/종료 시각을 입력받아 해당 시간대에만 캡처를 자동 실행하는 타이머 기능을 구현했다. |
| **Value Delivered** | Plan 요구사항 6개 항목 100% 달성. 야간/특정 시간대 자동 캡처 및 백그라운드 모드 연동 완료. |
| **Core Outcome** | 사용자가 자리를 비워도 지정 시간대 트래픽만 자동 수집 가능. 기존 테스트 전부 통과. |

---

## 1. 구현 결과

### 1.1 변경 파일

| 파일 | 변경 내용 |
|------|-----------|
| `internal/cli/prompt.go` | `Config`에 `TimerStart`, `TimerEnd` 필드 추가, `promptTimer()`, `promptTimeInput()` 함수 추가 |
| `internal/cli/runner.go` | `time` import 추가, 타이머 대기(`time.After`) 및 `context.WithDeadline` 적용 |

### 1.2 추가된 주요 코드

**prompt.go**
- `Config.TimerStart time.Time` / `Config.TimerEnd time.Time`
- `promptTimer()` — y/N 선택 후 시작/종료 시각 입력
- `promptTimeInput()` — HH:MM 파싱, 과거 시각은 내일로 자동 설정

**runner.go**
- 시작 시각 도달 전 `select { time.After / ctx.Done }` 대기
- 종료 시각 `context.WithDeadline`으로 자동 종료
